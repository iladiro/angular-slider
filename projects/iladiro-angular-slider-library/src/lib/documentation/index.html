Slider Documentation

Description

Slider is a reusable angular component. You can use the default template or pass your custom template following some configurations.

Default template
<slider-widget
    [slideList]="list">
</slider-widget>
Custom template
<slider-widget
    [slideList]="list">
    [slideWidth]="300"
    [paginatorArrowNext]="'fa fa-arrow-circle-o-right fa-4x'"
    [paginatorArrowPrev]="'fa fa-arrow-circle-o-left fa-4x'"
    [paginatorTemplate]="'templateOne'">
</slider-widget>
Custom dynamic template
<slider-widget
    [slideWidth]="300"
    [paginatorArrowNext]="'fa fa-arrow-circle-o-right fa-4x'"
    [paginatorArrowPrev]="'fa fa-arrow-circle-o-left fa-4x'"
    [paginatorTemplate]="'templateOne'"
    [slideList]="list">
    <ng-template
        #customTemplate
        let-slide="slide"
        let-first="first"
        let-last="last"
        let-index="index"
        let-activeIndex="activeIndex"
        let-direction="direction">
        <slider-video-card
            [slide]="slide"
            [index]="index"
            [activeIndex]="activeIndex"
            [first]="first"
            [last]="last"
            [direction]="direction">
        </slider-video-card>
    </ng-template>
</slider-widget>

You can choose which type of paginator to use, there are three different types.

USAGE

  • Import in your module SliderModule
  • Use slider selector to show it <slider-widget></slider-widget>

CONFIGURATION

Slider Widget accept only Slide object type array.

INTERFACE

Convert your array in an Slide object array. As you can see, not all properties are required.

interface Slide {
    link?: string;
    mediaSrc: string;
    tag?: string;
    title: string;
    description?: string;
    cta?: string;
    parag?: string;
}

Properties

Property Type Definition Default value Example 
slideList
 
Array

This widget expect a list of Slide to show

! If one of the non-mandatory properties is not passed in the object, the space assigned to that property in the view will not be redirected.

In ts file:
list: Slide[] = [
    {
        link: 'string',
        mediaSrc: 'string',
        tag: 'string',
        title: 'string 1',
        description: 'string',
        cta: 'string',
        parag: 'string'
    }
]

In html file:
<slider-widget [slideList]="list"></slider-widge> 
slideWidth
 
Number

You can change the width of the slides which will automatically be passed in px

455px <slider-widget [slideWidth]="300"></slider-widge> 
paginatorTemplate
 
String

You can choose which type of pagination you want to use between:

  • default
  • templateOne
  • templateTwo
 default <slider-widget [paginatorTemplate]="'templateOne'"></slider-widge> 
paginatorArrowStart
 
String

You can change the arrow that jumps to the first item in the list

 fa fa-angle-double-left <slider-widget [paginatorArrowStart]="'fa fa-arrow-circle-o-left fa-4x'"></slider-widge> 
paginatorArrowEnd
 
String

You can change the arrow that jumps to the last item in the list

 fa fa-angle-double-right <slider-widget [paginatorArrowEnd]="'fa fa-arrow-circle-o-left fa-4x'"></slider-widge> 
paginatorArrowNext
 
String

You can change the arrow that goes to the next item

 fa fa-chevron-right <slider-widget [paginatorArrowNext]="'fa fa-arrow-circle-o-right fa-4x'"></slider-widge> 
paginatorArrowPrev
 
String

You can change the arrow that moves to the previous item

 fa fa-chevron-left <slider-widget [paginatorArrowPrev]="'fa fa-arrow-circle-o-left fa-4x'"></slider-widge> 
paginatorMaxVisiblePages
 
Number

If you were to use the templateTwo template, you might need this option, i.e. it allows you to tell the paginator that you want to display maximum n numbers of pages

5 <slider-widget [paginatorTemplate]="'templateTwo'" [paginatorMaxVisiblePages]="5"></slider-widge> 
customClass
 
String

You can also pass a class or a list of classes to add to the parent slide tag. It can be useful, for example, to use the default card but customize it specifically for that section

<slider-widget [customClass]="'myClass'"></slider-widge>

Events

Event name Return Description Example 
goNextEvent
 
Object
Es. {page: 2, direction: 'next'}

Allows you to capture the event when the pager is used to move to the next slide

<slider-widget (goNextEvent)="console.log($event)"></slider-widget> 
goPrevEvent
 
Object
Es. {page: 2, direction: 'prev'}

Allows you to capture the event when the pager is used to move to the previous slide

<slider-widget (goPrevEvent)="console.log($event)"></slider-widget> 
goFirstEvent
 
Object
Es. {page: 1, direction: 'first'}

Allows you to capture the event when the pager is used to return to the first slide

<slider-widget (goFirstEvent)="console.log($event)"></slider-widget> 
goLastEvent
 
Object
Es. {page: 2, direction: 'last'}

Allows you to capture the event when the pager is used to go back to the last slide

<slider-widget (goLastEvent)="console.log($event)"></slider-widget>
